# Quant 项目学习指南

> 从这里开始，系统地理解一个真实运行的 A 股量化交易系统。

---

## 你可以学到什么

这个项目涵盖了量化交易从数据到执行的完整链路：

```
数据获取 → 因子挖掘 → 模型训练 → 回测验证 → 模拟盘 → Web监控
```

每个环节都有真实的代码、数据和踩坑记录。适合：
- 想做量化但不知道从哪开始的初学者
- 有编程基础想了解量化交易全貌的工程师
- 对 A 股规则策略感兴趣的研究者

## 学习路径

### 第一步：看懂策略（1-2 天）

先理解「涨停策略 E4」的完整逻辑——它是目前表现最好的策略，也是最容易理解的：

**阅读顺序：**
1. `CLAUDE.md` — 项目总览，了解架构和快速命令
2. `README.md` — 回测结果和策略对比
3. `scripts/run_daily_paper_lu.py` — 涨停模拟盘脚本（注释清晰，从头读到尾）
4. 回测结果：跑一遍 `python scripts/backtest_limit_up_strategy.py --start 2020-01-01 --mcap-proxy --top-n 5 --min-conditions 4 --exit-stop 0.08`

**核心问题（试着回答）：**
- 为什么只选 Top-5 而不是 Top-20？
- 8% 止损是「保护伞」还是「割肉刀」——数据怎么说？
- 为什么去跌停比含跌停更好？

### 第二步：理解回测（2-3 天）

回测是量化的心脏。错误回测 = 纸上富贵。

**阅读顺序：**
1. `scripts/backtest_limit_up_strategy.py` — 涨停策略回测
2. 重点看这几段代码：
   - `run_screening()` — 每天怎么筛选股票
   - 次日收益计算（T+1 执行，open→close）
   - 成本扣除（佣金+印花税+滑点）
   - Sharpe/MDD/年化收益的计算
3. `models/dataset.py` — Walk-Forward 数据切分
4. `tests/test_overfit_check.py` — 过拟合检测

**关键概念：**
- **前视偏差 (Look-ahead Bias)**：回测时不能用到未来的信息。比如 T 日选股时，不能知道 T+1 的涨跌
- **滑点 (Slippage)**：真实成交价和回测假设价的差距。小票滑点更大
- **幸存者偏差 (Survivorship Bias)**：用今天的股票池回测过去，会漏掉已退市的票

**核心问题：**
- 这个策略的回测收益中有多少来自「涨停因子」本身，多少来自数据偏差？
- 为什么不同回测区间（2015-2019 vs 2020-2026）结果差异巨大？

### 第三步：模拟盘 vs 回测的差距（1-2 天）

回测中的策略搬到实盘，效果通常会打折。模拟盘是中间的过渡站。

**阅读顺序：**
1. `portfolio/paper_engine.py` — PaperEngine 完整实现
2. `scripts/run_daily_paper_lu.py` 的 `execute()` 函数
3. 对比回测结果和模拟盘真实盈亏（本项目的模拟盘跑了约两周，权益曲线见 Web 页面）

**关键差距来源：**
- **T+1 执行**：回测用的是「次日开盘价」，模拟盘真实等到次日开盘才能成交
- **数据延迟**：收盘后数据不是立即就绪的，可能导致执行偏差
- **流动性和成交**：回测假设总能成交，真实市场可能买不到或卖不掉
- **市场状态**：回测涵盖了牛熊周期，模拟盘可能赶上一段连续下跌（比如最近两周）

**核心问题：**
- 如果模拟盘连续亏损，如何判断是「策略失效」还是「短期波动」？
- 什么信号会让你决定停掉一个模拟盘策略？

### 第四步：实验思维（持续）

这个项目最重要的不是代码，而是**实验驱动的方法论**。

**我们做过的实验（每次都有对照和结论）：**

| 实验 | 方法 | 结论 |
|------|------|------|
| 跌停过滤有用吗？ | 含 vs 去，分年对比 | 5/7年去跌停更好 |
| 自适应跌停？ | 按波动率动态切换 | 不可行，whipsaw |
| 开盘Gap过滤？ | 跳低开不买 vs 照常买 | 损害收益 |
| 移动止盈？ | 5%/8%/10%回撤卖 | 全部降低收益 |
| 破MA5卖？ | 跌破均线就卖 | 灾难性结果 |
| 黑名单？ | 人工排除特定股票 | 去跌停已足够 |

**核心教训：**
- **直觉常常错**。你以为加保护会更好，数据说不是
- **勤回测，不瞎猜**。每个想法都先用 6 年数据验证
- **理解为什么**。不只是看数字，要解释背后的市场逻辑

## 关键概念速查

### 涨停策略四大条件

| 条件 | 含义 | 为什么重要 |
|------|------|-----------|
| 市值 30-500亿 | 中小盘 | 太大涨不动，太小是庄股 |
| MA5 > MA10 | 短期趋势向上 | 只做上升趋势，不抄底 |
| 近20日涨停>1次 | 有动量 | 涨停是A股最强的动量信号 |
| ~~近10日无跌停~~ | （已删除） | 跌停多为情绪错杀，过滤掉错过反弹 |

### 风控体系

```
第一层：个股 8% 止损     → 单票跌 8% 无条件卖
第二层：组合 20% 回撤减仓 → 总资产从高点跌 20%，仓位减半
第三层：组合 25% 回撤清仓 → 总资产跌 25%，全部卖出
```

### 收益来源解剖

涨停策略的盈利不是靠高胜率，而是靠**非对称收益**：

```
60% 的交易：亏 2-5%（频繁的小止损）
30% 的交易：赚 5-15%（正常轮动收益）
10% 的交易：赚 30-80%（抓到连板股！）

→ 一只大赚票覆盖 10-15 只小亏票
→ 年化 300%+ 就是这么来的
```

这也解释了为什么「移动止盈」和「逃顶」会降低收益——**你截断了那 10% 的大赢家**。

## 常见坑

| 坑 | 表现 | 怎么发现 |
|----|------|---------|
| 前视偏差 | 回测好得离谱 | 逐日检查数据对齐，确认没有未来信息 |
| 滑点估计错误 | 小票回测高但实盘差 | 小票用 0.3% 滑点，大票用 0.1% |
| T+0 而不是 T+1 | 模拟盘当天买卖 | A股必须 T+1，回测用次日开盘价算收益 |
| 市值数据不足 | 早期回测缺市值信息 | 用 `--mcap-proxy`（隐含股本代理） |
| run_id 冲突 | 两个策略写同一张表 | 各自独立 run_id（1/2/3） |
| 信号重复写入 | 同一天信号有重复 rank | ON CONFLICT DO UPDATE 去重 |
