# 板块轮动策略 — 完整设计方案

> 创建: 2026-06-19 | 状态: 方案打磨完成，待 7 月初实现 | 目标平台: Windows + AmazingData

---

## 一、策略定位

在现有「小市值反转（主力）+ 妖股（卫星）」之外，新增第三条腿——**板块轮动**。

核心逻辑：
- 宏观层判断「哪些大类在变热」→ 锁定方向
- 微观层判断「大类下哪些细分在低迷」→ 找入场时机
- 热度上来后卖出 → 在同大类内轮换到下一个低迷细分

---

## 二、两层架构

```
仓位: 50-80% 核心轮动 + 20-50% 卫星仓（主观择时）

═══════════════════════════════════════════════════
宏 观 层 — 大类资产配置（每周打分，周五盘后）
═══════════════════════════════════════════════════

6 大类映射:
  科技 = 电子 + 计算机 + 通信 + 传媒
  周期 = 有色 + 钢铁 + 化工 + 采掘 + 建材
  消费 = 食品饮料 + 家电 + 纺织服装 + 汽车 + 商贸
  金融 = 银行 + 非银 + 地产
  医药 = 医药生物
  制造 = 机械 + 电气设备 + 国防军工 + 电力设备

打分（截面 rank 标准化）:
  残差动量 = 行业指数收益 - β × 市场收益       权重 40%
  广度     = 成分股站上 MA20 的比例             权重 30%
  资金流   = 放量上涨股占比                      权重 30%

过滤规则:
  → 拥挤度（换手率分位 > 90%）的大类直接排除
  → 选 Top 2-3 大类进入微观层

═══════════════════════════════════════════════════
微 观 层 — 细分行业/概念轮动（每周打分）
═══════════════════════════════════════════════════

标的: 申万二级行业（131个）+ 概念板块（423个）
      仅在宏观层选中的大类范围内扫描

三轴打分:
  趋势 (40%) = 20日动量截面 rank
  反转 (35%) = 1 - 恐慌贪婪指数 / 100
  安全 (25%) = 1 - 换手率历史分位（低拥挤 = 高分）

信号规则:
  🟢 买入: 总分 > 60% 且 恐惧贪婪 < 40
  🔴 卖出: 恐惧贪婪 > 70 或 拥挤度 > 90%
  ⏱ 冷却: 卖出后 5 交易日不买回同一细分
  🚫 一票否决: 板块情绪 > 85% 分位 → 直接排除

仓位约束:
  单细分 ≤ 20%
  同大类内高相关板块（20日相关性 > 0.8）合并仓位上限
  大类全线过热 → 保留 30% 现金
```

## 六、回测设计要点

- **独立脚本** `bt_sector_rotation.py`，遵循 CLAUDE.md 规则 4
- **无未来函数**: 所有因子 T 日收盘计算，T+1 日开盘价执行
- **统一参数**: 复用 `TradingConfig`（佣金万0.9、印花税万5、滑点0.1%、初始100万）
- **输出**: 交割单 CSV `trades_sector_rotation_{topn}_{start}_{end}_{label}.csv`
- **如果用 ML**: walk-forward 滚动训练，严格时间分割，无过拟合

## 七、风险与边界

| 风险 | 应对 |
|------|------|
| 概念板块成分股不稳定 | 每周重算成分股，不依赖历史快照 |
| 行业轮动在熊市失效 | 加市场状态判断，大盘 < MA60 时空仓 |
| 合成收益不准确 | 7月 AmazingData 接入后改用真实指数 |
| 单板块容量不足 | 单细分 ≤ 20% 仓位 + 市值过滤（> 30亿） |
| 过拟合 | 简单规则优先，ML 仅做辅助且 walk-forward 验证 |

## 八、备选优化方向（实验阶段）

1. **扩散动量**（国海证券）: 广度 × 强度代替简单动量
2. **HMM 市场状态识别**: 3 状态（牛/熊/震荡），不同状态不同参数
3. **残差动量**: 剥离市场 beta 后的纯行业 alpha
4. **LLM 舆情打分**: 用新闻/研报文本增强板块热度判断
5. **CTA 信号辅助**: 行业轮动选方向 + CTA 定买卖时点
